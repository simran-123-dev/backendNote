# 📖 Important Questions & Answers

---

## ❓ What is Node.js Architecture?

### ✅ Answer

Node.js follows a **Single-Threaded, Event-Driven, Non-Blocking I/O Architecture**.

Instead of creating a new thread for every client request, Node.js uses a **single main thread** along with an **Event Loop** to handle multiple client requests efficiently.

When a request takes time (like reading a file or querying a database), Node.js delegates it to the system (or thread pool) and continues processing other requests. Once the operation is complete, the Event Loop receives the result and sends the response.

### Architecture Diagram

```text
          Client Requests
                 │
                 ▼
        ┌────────────────┐
        │   Event Queue  │
        └────────────────┘
                 │
                 ▼
        ┌────────────────┐
        │   Event Loop   │
        └────────────────┘
                 │
      ┌──────────┴──────────┐
      │                     │
Blocking Task?         Non-Blocking Task
      │                     │
      ▼                     ▼
 Wait for Task      Thread Pool / OS
      │                     │
      └──────────┬──────────┘
                 ▼
             Send Response
```

### Key Points

- Single-threaded architecture
- Event-driven programming
- Uses an Event Loop
- Supports asynchronous operations
- High performance for I/O-intensive applications

## ❓ What is a Blocking Request?

### ✅ Answer

A **Blocking Request** is a request that **stops the execution of the program until the current task is completed**.

During this time, no other request can be processed.

### Example

```javascript
const fs = require("fs");

const data = fs.readFileSync("data.txt", "utf8");

console.log(data);
console.log("Program Finished");
```

### Execution Flow

```text
Read File
    │
    ▼
Wait Until Reading Completes
    │
    ▼
Print File Data
    │
    ▼
Program Finished
```

### Characteristics

- Synchronous
- Blocks the Event Loop
- Slower for multiple users
- Easy to understand

---

## ❓ What is a Non-Blocking Request?

### ✅ Answer

A **Non-Blocking Request** allows the program to continue executing other tasks without waiting for the current operation to finish.

The task runs in the background, and a callback executes once it is complete.

### Example

```javascript
const fs = require("fs");

fs.readFile("data.txt", "utf8", (err, data) => {
    console.log(data);
});

console.log("Program Finished");
```

### Execution Flow

```text
Start Reading File
        │
        ▼
Continue Executing Next Line
        │
        ▼
Program Finished
        │
        ▼
File Reading Completes
        │
        ▼
Display File Data
```

### Characteristics

- Asynchronous
- Does not block the Event Loop
- Better performance
- Handles multiple requests efficiently
- Preferred in Node.js applications

---

## ❓ Difference Between Blocking and Non-Blocking Requests

| Blocking Request | Non-Blocking Request |
|------------------|----------------------|
| Synchronous | Asynchronous |
| Waits for task completion | Continues execution immediately |
| Blocks the Event Loop | Does not block the Event Loop |
| Slower for concurrent users | Faster and scalable |
| Example: `readFileSync()` | Example: `readFile()` |

## 🎯 Interview Questions

### Q1. Why is Node.js called Single-Threaded?

**Answer:** Because it uses a single main thread to execute JavaScript code while the Event Loop manages asynchronous operations.

---

### Q2. What is the Event Loop?

**Answer:** The Event Loop continuously checks for completed asynchronous tasks and executes their callback functions without blocking the main thread.

---

### Q3. Why is Node.js fast?

**Answer:** Node.js is fast because it uses Google's V8 JavaScript Engine and a non-blocking, event-driven architecture.

---

### Q4. Which is better: Blocking or Non-Blocking?

**Answer:** Non-blocking operations are preferred because they improve performance and allow Node.js to handle many client requests simultaneously.

---

## 📝 Quick Revision

- Node.js uses a **Single-Threaded Event-Driven Architecture**.
- The **Event Loop** handles asynchronous operations.
- **Blocking** operations wait until completion.
- **Non-Blocking** operations run in the background.
- Non-blocking I/O makes Node.js fast and scalable.
